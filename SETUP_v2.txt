AUTOPILOT V2.0 — CONTINUOUS IMPROVEMENT WITH MODEL SWITCHING
═══════════════════════════════════════════════════════════════════

UPGRADE SUMMARY
───────────────

❌ OLD: Discrete 2-hour cycles (cronjob every 2h → 1 cycle → sleep 2h)
✅ NEW: Continuous 2-5 min cycles (loop forever → adaptive sleep)

KEY IMPROVEMENTS
────────────────

1. RUNS FOREVER — No more waiting 2 hours between cycles
   Instead of: Cron tick @ 14:00 → cycle → sleep 120m
   Now: Analyze (60s) → Plan (30s) → Execute (90s) → Test (120s) → Commit (10s) → sleep 60s → repeat

2. MODEL SWITCHING PER-PHASE — Each phase uses its optimal model
   Phase 1 (Analyze):     gpt-4o / openrouter  (128k context for deep audit)
   Phase 2 (Plan):        deepseek-v4-flash / opencode (fast planning)
   Phase 3 (Execute):     agy-switch / opencode (+ automatic fallback chain)
   Phase 4 (Test):        gpt-4o-mini / opencode (cost-optimized)
   Phase 5 (Commit):      deepseek-v4-flash / opencode (lightweight)
   Phase 6 (Report):      gpt-4o-mini / opencode (cost-optimized)

3. RATE-LIMIT RECOVERY — agy-switch automatically falls back
   If rate-limited on deepseek:
   Attempt 1: deepseek-v4-flash-free (opencode)
   Attempt 2: gpt-4o-mini (opencode) 
   Attempt 3: gpt-4o (openrouter)
   Attempt 4: claude-opus-4 (anthropic)
   Attempt 5: qwen2.5-72b (openrouter)
   → No user intervention needed, fully transparent

4. ADAPTIVE CYCLE TIMING
   No issues found (0 changes):  Sleep 180s (reduce check frequency)
   Minor issues (1-3 tasks):     Sleep 120s (standard)
   Major issues (4+ tasks):      Sleep 60s (urgent mode)
   Test failures:                Sleep 30s (critical)

      execute:   { model: "agy-switch", provider: "opencode" }
      test:      { model: "gpt-4o-mini", provider: "opencode" }

Then restart daemon for changes to take effect.

Edit agy-switch fallback chain:
  vi ~/.local/bin/agy-switch
  
  FALLBACK_CHAIN=(
    "gpt-4o-mini:opencode"
    "gpt-4o:openrouter"
    "claude-opus-4:anthropic"
    "qwen2.5-72b:openrouter"
  )

TROUBLESHOOTING
────────────────

Q: Daemon hangs on Phase 1 (Analyze)
A: gpt-4o might be slow. Check OpenRouter status:
   $ agy-switch --model gpt-4o --provider openrouter -p "test"
   Consider switching analyze phase to a faster model.

Q: Phase 3 keeps falling back through all models
A: All providers rate-limited. Wait 60s or add another provider.
   Edit FALLBACK_CHAIN in ~/.local/bin/agy-switch

Q: No changes detected (consecutive_no_changes > 5)
A: Codebase already clean! Reduce cycle frequency to save compute.
   Edit daemon sleep logic.

Q: Discord webhook failing
A: Verify webhook URL in config:
   $ curl -X POST <WEBHOOK_URL> -H "Content-Type: application/json" -d '{"content":"test"}'
   Should return HTTP 204 No Content.

Q: Git push failing
A: Requires GitHub auth token:
   $ gh auth login --with-token < ~/.github-token
   Or configure remote with embedded token.
