FILES TO USE
────────────

1. START DAEMON:
   ~/.local/bin/autopilot-continuous-daemon
   
   Command:
   nohup autopilot-continuous-daemon > logs/daemon.log 2>&1 &
   
   What it does:
   - Loops forever running 6-phase cycles (Analyze→Plan→Execute→Test→Commit→Report)
   - Each phase uses optimal model (configurable)
   - Sleeps 60-180s between cycles (adaptive)
   - Posts to Discord webhook after each cycle
   - Commits to git with conventional messages

2. MODEL SWITCHING:
   ~/.local/bin/agy-switch
   
   Command:
   agy-switch -p "refactor this function" --model gpt-4o --provider openrouter
   
   What it does:
   - Wraps agy with explicit model/provider selection
   - Falls back transparently if rate-limited
   - Fallback chain: deepseek → gpt-4o-mini → gpt-4o → claude → qwen2.5-72b
   - Used in Phase 3 (Execute) of the daemon

3. PHASE MODEL CONFIG:
   ~/.hermes/profiles/autopilot-worker/config.yaml
   
   Section: agent.autopilot_phase_models
   
   Current setup:
   - analyze:   gpt-4o / openrouter       (128k context, deep audit)
   - plan:      deepseek-v4-flash / opencode (fast, cheap)
   - execute:   agy-switch / opencode     (auto-fallback to 4 other models)
   - test:      gpt-4o-mini / opencode    (cost-optimized)
   - commit:    deepseek-v4-flash / opencode (lightweight)
   - report:    gpt-4o-mini / opencode    (cost-optimized)

═══════════════════════════════════════════════════════════════════════════════

CYCLE TIMING
────────────

TOTAL CYCLE DURATION: 5-12 minutes (depends on workload)

Breakdown:
  Phase 0 (Bootstrap):  30-60s   (setup, check git status)
  Phase 1 (Analyze):    60-120s  (scan codebase with gpt-4o)
  Phase 2 (Plan):       20-40s   (fast planning with deepseek)
  Phase 3 (Execute):    90-180s  (code edits via agy-switch)
  Phase 4 (Test):       60-120s  (pytest + ruff linting)
  Phase 5 (Commit):     5-15s    (conventional commit)
  Phase 6 (Report):     2-5s     (summary + Discord)

SLEEP BETWEEN CYCLES (adaptive):
  - 0 changes found:    180s (reduce frequency, codebase clean)
  - 1-3 changes:        120s (standard, minor fixes)
  - 4+ changes:         60s  (urgent mode, many issues)
  - Test failures:      30s  (critical, retry fast)

EFFECTIVE RATE: One complete cycle every 5-12 minutes (forever)

═══════════════════════════════════════════════════════════════════════════════

AGY-SWITCH FALLBACK MECHANISM
──────────────────────────────

When agy-switch (Phase 3) receives a rate-limit error:

  Attempt 1: deepseek-v4-flash-free / opencode
    ↓ (if rate-limited)
  Attempt 2: gpt-4o-mini / opencode
    ↓ (if rate-limited)
  Attempt 3: gpt-4o / openrouter
    ↓ (if rate-limited)
  Attempt 4: claude-opus-4 / anthropic
    ↓ (if rate-limited)
  Attempt 5: qwen2.5-72b / openrouter
    ↓ (if rate-limited)
  FAIL: All models exhausted

USER EXPERIENCE: Silent retry. User sees one log line per retry.
              No manual intervention needed.

EDIT FALLBACK CHAIN:
  vi ~/.local/bin/agy-switch
  Search: FALLBACK_CHAIN=(
  Edit the array to add/remove providers

TROUBLESHOOTING
────────────────

Q: Daemon exits immediately
A: Check logs: tail -f ~/Aarav/Autopilot/logs/daemon.log
   Common: DISCORD_WEBHOOK not set, Hermes not installed, PATH issues

Q: agy-switch always falls back to qwen2.5-72b
A: All providers rate-limited. Options:
   1. Wait (most providers reset every hour)
   2. Add another provider to FALLBACK_CHAIN
   3. Reduce cycle frequency (increase sleep duration)
   4. Run daemon foreground and check errors: autopilot-continuous-daemon

Q: Discord webhook gets HTTP 401/403
A: Invalid webhook URL. Verify:
   curl -X POST "https://discord.com/api/webhooks/YOUR/URL" \
     -H "Content-Type: application/json" \
     -d '{"content":"test"}'
   Should return 204. If 401/403, URL is wrong or expired.

Q: Phase 1 (Analyze) takes >2 minutes
A: gpt-4o might be slow. Options:
   1. Switch to faster model: vi config.yaml → change analyze model
   2. Reduce context: edit daemon to send smaller code snippets
   3. Check OpenRouter quota: agy-switch -p "test" --model gpt-4o

Q: No commits appearing in git
A: Check daemon logs for git errors:
   grep -i "git\|commit" ~/Aarav/Autopilot/logs/daemon.log
   Verify git repo is clean: cd ~/Aarav/Autopilot && git status

Q: Daemon uses too much CPU
A: Phases running too fast. Options:
   1. Increase sleep duration between cycles
   2. Reduce number of tasks planned per cycle
   3. Use cheaper/faster models in config.yaml
