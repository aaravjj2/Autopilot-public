### Reliability Issues (2)

**4. [RELIABILITY] `autopilot-continuous.py` RATE_LIMITS are in-memory only**
- Lines 24-30: The `RATE_LIMITS` dict lives entirely in memory, never persisted to disk
- Process restart → all cooldowns reset → immediately hammers rate-limited APIs
- `autopilot.py` already has working disk persistence via `cd_write()` — continuous loop should match

**5. [RELIABILITY] `src/apex/core/retry.py` has no jitter or exponential backoff**
- Line 54: `time.sleep(max(0.0, float(backoff_seconds)))` — flat, deterministic sleep every retry
- When called by 7+ consumers (broker, execution, ingestion, engine), creates thundering-herd timing
- Fix: Add `random.uniform(0.8, 1.2) * backoff_seconds * (2 ** attempt)` for exponential backoff with jitter
